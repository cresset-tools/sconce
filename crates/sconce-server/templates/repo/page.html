CI access (OIDC)

Zero-secret CI: a workflow POSTs its platform OIDC JWT and a matching policy mints a short-lived token (no stored secret). A read policy (exchange at {{ base }}/oauth/ci) mints a serving token; a publish policy (exchange at {{ base }}/oauth/ci-publish) mints a publish token so sconce publish can push package versions.
